## 一、什麼是 Big O？ (What is Big O Notation?)

**Big O 記號**是描述演算法在輸入規模 $n$ 增大時，**執行時間或記憶體用量的成長速度**。

- 它不代表「精確的執行時間」，而是「**成長趨勢**的上界」
- 省略常數與低次項：$3(n^2) + 5n + 10$ → $O(n^2)$
- 目的：在不同機器、不同語言間，**公平比較演算法的效率**

```
n = 1,000 時各複雜度的大約運算次數：

O(1)         →           1  次  （常數，最快）
O(log n)     →          10  次
O(n)         →       1,000  次
O(n log n)   →      10,000  次
O(n²)        →   1,000,000  次
O(2ⁿ)        →  2^1000 次  （指數，不可行）
```

---

## 二、三種情況的分析

同一個演算法在不同的「輸入狀態」下，時間複雜度可能不同：

| 情況 | 說明 | 分析意義 |
|------|------|---------|
| **最佳情況 Best Case** | 最幸運的輸入（如陣列已排序） | 理論下限，實際很少達到 |
| **平均情況 Average Case** | 隨機輸入的期望時間 | 最能反映真實效能 |
| **最壞情況 Worst Case** | 最不幸的輸入（如逆序陣列） | **系統設計最重視**，確保上限 |

## 三、各排序演算法的 Big O 深度解析

### 3.1 選擇排序 Selection Sort — $O(n^2)$

**計算邏輯：**

第 1 輪：比較 $n-1$ 次，找最小值  
第 2 輪：比較 $n-2$ 次  
…  
第 $n-1$ 輪：比較 $1$ 次

$$T(n) = (n-1) + (n-2) + \cdots + 1 = \frac{n(n-1)}{2} = O(n^2)$$

**關鍵特點：不管輸入狀態，永遠是 $O(n^2)$**  
→ 因為每輪都必須掃完整個未排序部分才能確認最小值。

```
n=5 的比較次數：4 + 3 + 2 + 1 = 10 次
n=10 的比較次數：9 + 8 + ... + 1 = 45 次
n 翻倍 → 比較次數約翻 4 倍（n² 的特徵）
```

---

### 3.2 插入排序 Insertion Sort — $O(n)$ ~ $O(n^2)$

**最佳情況 $O(n)$：陣列已排序**

每次取出新元素，發現它 ≥ 左邊所有元素，只需 1 次比較就能插入。  
共 $n-1$ 次插入，每次 1 次比較 → $O(n)$

```
已排序輸入：[1, 2, 3, 4, 5]
每次 while 條件第一下就失敗，不進入迴圈
比較次數 = n-1 次 → O(n)
```

**最壞情況 $O(n^2)$：陣列逆序排列**

每次取出新元素，必須把前面「所有」已排序元素都向右移。

$$T(n) = 1 + 2 + 3 + \cdots + (n-1) = \frac{n(n-1)}{2} = O(n^2)$$

```
逆序輸入：[5, 4, 3, 2, 1]
插入 4：移動 1 次
插入 3：移動 2 次
插入 2：移動 3 次
插入 1：移動 4 次
總計：1+2+3+4 = 10 次（n=5）
```

**結論：** 插入排序對「幾乎已排序」的資料非常有效率。

---

### 3.3 合併排序 Merge Sort — $O(n \log n)$（三種情況相同）

**為什麼是 $\log n$？**

每次對半切分，切到長度為 1 需要：

$$n \to \frac{n}{2} \to \frac{n}{4} \to \cdots \to 1$$

共 $\log_2 n$ 層

**每層的總合併工作量？**

每一層要合併的元素總數都是 $n$，每層花 $O(n)$。

$$T(n) = \underbrace{O(n)}_{\text{第1層}} + \underbrace{O(n)}_{\text{第2層}} + \cdots + \underbrace{O(n)}_{\log n \text{ 層}} = O(n \log n)$$

```
n = 8 的分層示意：

層 1：[8個] → 合併工作量 8
層 2：[4個][4個] → 合併工作量 4+4=8
層 3：[2][2][2][2] → 合併工作量 2+2+2+2=8
層 4：[1][1][1][1][1][1][1][1] → 底層（基底條件）

共 log₂(8) = 3 層，每層 8 次比較 → 8 × 3 = 24 次
```

**不管輸入狀態都是 $O(n \log n)$**：因為分治過程固定，不受資料順序影響。

---

### 3.4 快速排序 Quick Sort — 平均 $O(n \log n)$，最壞 $O(n^2)$

**平均情況 $O(n \log n)$：Pivot 每次約平均分割**

若 Pivot 每次都把陣列約對半分，分割層數 ≈ $\log n$，每層比較 $O(n)$ 次：

$$T(n) = O(n \log n)$$

**最壞情況 $O(n^2)$：Pivot 每次選到最大或最小值**

```
陣列已排序 [1, 2, 3, 4, 5]，選末端為 pivot：
第1輪：pivot=5，左邊 [1,2,3,4]，右邊 []  → n-1 次比較
第2輪：pivot=4，左邊 [1,2,3]，右邊 []   → n-2 次比較
...
T(n) = (n-1) + (n-2) + ... + 1 = O(n²)
```

**為什麼平均好過最壞？**

隨機輸入時，Pivot 平均落在中間附近的機率遠大於極端的機率，期望值為 $O(n \log n)$。

**改善策略：**
- **三數取中 (Median-of-Three)**：取首、中、末三者的中位數為 Pivot
- **隨機化 Pivot**：隨機選取，使最壞情況在實際中幾乎不可能

---

### 3.5 堆積排序 Heap Sort — 恆定 $O(n \log n)$

**建堆階段 (Heapify)：$O(n)$**

直覺上以為要 $O(n \log n)$，但由於底層節點多而高度低，數學分析可證明建堆是 $O(n)$。

**取出階段：$O(n \log n)$**

每次取出最大值後需 siftDown，高度為 $O(\log n)$，共取 $n$ 次：

$$T = O(n) + n \times O(\log n) = O(n \log n)$$

**特點：** 空間只需 $O(1)$（原地排序），且最壞情況也保證 $O(n \log n)$，這點優於快速排序。

## 四、Big O 成長速度直觀比較

```
比較次數（n = 16）：

O(1)         =        1 次
O(log n)     =        4 次   （log₂ 16 = 4）
O(n)         =       16 次
O(n log n)   =       64 次
O(n²)        =      256 次
```

### 成長速度視覺化

```
次數
  ↑
256 |                              ●  O(n²)
    |
 64 |                   ●            O(n log n)
    |
 16 |          ●                     O(n)
    |
  4 |    ●                           O(log n)
  1 | ●                              O(1)
    +——————————————————————————————→ n=16
```

## 五、空間複雜度（輔助記憶體）

| 演算法 | 空間複雜度 | 原因 |
|--------|-----------|------|
| 選擇排序 | $O(1)$ | 原地排序，只用少量暫存變數 |
| 插入排序 | $O(1)$ | 原地排序 |
| 合併排序 | $O(n)$ | 合併時需要大小為 $n$ 的暫存陣列 |
| 快速排序 | $O(\log n)$ | 遞迴呼叫堆疊深度平均 $\log n$ 層 |
| 堆積排序 | $O(1)$ | 原地排序（使用陣列表示堆積） |

## 六、如何選擇排序演算法？

```
輸入規模 n 很小（< 20）?
  → 插入排序（常數小，實作簡單）

資料幾乎已排序?
  → 插入排序（最佳情況 O(n)）

需要穩定排序?
  → 合併排序

記憶體緊張，必須原地排序?
  → 堆積排序（O(n log n) + O(1) 空間）

一般用途，追求平均效能?
  → 快速排序（常數因子小，快取友好）

保證最壞情況也是 O(n log n)?
  → 合併排序 或 堆積排序（不要用快速排序）
```

## 七、學習筆記摘要

```
Big O 排序核心要點：
┌──────────────────────────────────────────────────────────────┐
│ ✦ O(n²)：Selection/Insertion（最壞）— 每輪掃描剩餘元素      │
│ ✦ O(n log n)：Merge/Heap（所有情況）— 分治/堆積操作         │
│ ✦ Quick Sort：平均 O(n log n)，最壞 O(n²)，最常用           │
│ ✦ Insertion Sort 最佳 O(n)：幾乎排好的資料最適合            │
│ ✦ 空間複雜度：Merge 需 O(n)，其餘可 O(1) 原地排序           │
│ ✦ n 翻倍時：O(n²) 增 4 倍，O(n log n) 增約 2 倍多          │
└──────────────────────────────────────────────────────────────┘
```
